# PRD.md — Product Requirements Document
# StreamVibe — Video Streaming Platform

> **One-line pitch:** A simple video-sharing platform where users upload, stream, and discover videos — like Instagram meets YouTube, powered by ImageKit.io + MongoDB Atlas.

---

## 1. WHAT WE ARE BUILDING

A web app where:
- Users register / login with email + password
- Users upload videos (MP4 files) to ImageKit.io
- Videos are stored as metadata in MongoDB Atlas
- Anyone logged in can watch videos streamed via ImageKit.io's HLS pipeline
- Users can like and follow other users
- A feed shows recent/popular videos

---

## 2. FEATURES IN SCOPE

### Auth
- Register with username, email, password
- Login / logout
- JWT-based session (stored in httpOnly cookie)
- Password hashed with bcrypt

### Video Upload
- Upload MP4 from browser directly to ImageKit.io via server-signed upload URL
- Store video metadata in MongoDB (title, description, tags, uploader, imagekit fileId, imagekit URL)
- Show upload progress bar
- Thumbnail auto-generated by ImageKit.io

### Video Feed (Home Page)
- List of all videos, newest first
- Each card shows: thumbnail, title, uploader username, view count
- Infinite scroll (load 12 at a time)

### Video Player Page
- HLS streaming via Video.js player fed the ImageKit.io `.m3u8` URL
- Shows title, description, uploader, like count
- Like button (toggle)
- View count increments on load

### User Profile Page
- Shows username, avatar (optional), list of uploaded videos
- Follow / Unfollow button
- Follower count + following count

### Search
- Search by video title (simple text search on MongoDB)

---

## 3. OUT OF SCOPE (DO NOT BUILD)

- Comments
- DMs / messaging
- Live streaming
- Subscriptions / payments
- Notifications (email or push)
- Video editing or cropping tools
- Mobile app (React web only)
- Admin dashboard
- Recommendation algorithm
- Ad system
- Multi-language / i18n
- Dark mode toggle (dark by default, period)

---

## 4. USERS

Single user type: authenticated users. No admin role needed.  
Unauthenticated users can browse the feed and watch videos but cannot upload or like.

---

## 5. SUCCESS CRITERIA

- User can register, login, upload a video, and stream it in under 5 minutes
- Video playback starts within 3 seconds (HLS adaptive bitrate via ImageKit.io)
- All API responses < 300ms (excluding video upload/stream)
- Works on Chrome and Firefox desktop
